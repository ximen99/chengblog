1. Introduction¶

Background for this report

In view of the progress of my research in developing quantative investment algorithm, this report can be seen as a sequel to the resistance and support index report, with the ideology behind that report and my early algorithms developed being to interpret the most famous traditional investment strategies, which can be either technical or fundamental, as a set of rules that computer can follow and are easy for us to evaluate through back testing. As Im scratching my head trying to come up with a better strategy to outperform the market, it dawned on me that if this is essentially a statistical problem, maybe statistical modeling can find some hidden rules which are not easy to grasp by human intuition and theres no better tool in terms of finding patterns than machine learning and AI, which are now so easily available thanks to the great open source community. Even though I had academic training in econometrics and experience in building logistic model to predict the likelihood of default in auto loans during my first job, it still took me a while to go through and get familiar with all the major ML algorithms including SVM, Ensemble Tree Models, KNN, PCA, DNN and etc. Through further research I find this a far more promising direction then my old way of trying to find a good strategy by inspiration. This report is one of the results in my attempts to use statistical model to find hidden rules in financial market.

HMM model

I came to the news that HMM is one of the main models used by the most successful quantative fund Renaissance Institutional Equities Fund. Through further studys I found out that the model makes perfect sense in the stock market. I'll give a short description of my understanding of HMM model here. HMM is stemed from Marcov Chain model, which we uses unintentionally in our daily life all the time, such as when we are trying to bet on tail in flip coin bet, when it has been head the first 10 times before this. (see this article). However when you see the tail comes up over 5 times you might think there's more chance that the game is rigged. That's the HMM way of thinking. In financial market, instead of trying to predict the future return, the better question to ask is what kind of market are we in right now. And we can use the features as the observations and the market condition as hidden state in HMM model. HMM model will tell us what market condition we are in by going through various technical indicators. It's so close to how a technical investor formulate it's trading strategy.

I used HMM model with Gaussian distribution with four hidden states. To save the time I'll skip trough the feature selection stage, which is basicly like how Edison invented light bulb by tring out different filament.

3. Strategy Development¶

Now it's time to create a trading strategy based on our model and conduct backtest. The function below returns whether or not we should hold position based on the model prediciton. The basic idea is that we don't want to make trading decision in dull market represented by green and red dots unless the slow bull or slow bear has been persisted for over 7 days out of 10. And if a directional market represented by orange and blue dots appear, we should be sure to update our holding status according to predicted market condition.

In [19]:
def check_hold(labels,hold_status):
    # check inital holding status
    if hold_status == None:
        if labels[-1] in ['bull','slow bull']:
            return 1
        else:
            return 0
    else:
        if hold_status == 0:
            if labels[-1] == 'bull':
                return 1
            elif (sum(np.in1d(labels,['bull','slow bull']))/len(labels))>0.7:
                return 1
            else:
                return 0
        if hold_status == 1:
            if labels[-1] == 'bear':
                return 0
            elif (sum(np.in1d(labels,['bear','slow bear']))/len(labels))>0.7:
                return 0
            else:
                return 1

labels = hmm.predict(X)
labels = pd.Series(labels).map(dic)
buy_ls = []
n = 10
for i in range(len(labels)):
    if i-n<0:
        if i == 0:
            buy_ls.append(check_hold(labels[0:i+1].values,None))
        else:
            buy_ls.append(check_hold(labels[0:i+1].values,buy_ls[-1]))
    else:
        buy_ls.append(check_hold(labels[i-n:i+1].values,buy_ls[-1]))
